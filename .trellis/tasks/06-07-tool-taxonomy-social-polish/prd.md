# 优化工具分类与社保功能

## Goal

Make the tool navigation feel more intentional and product-like, then continue polishing the income and social-insurance tools with small, practical improvements that fit the existing Liquid Glass interface.

## What I already know

* The user called out the sidebar/tab taxonomy as inelegant, especially the current "收入与社保" category relative to the other groups.
* The app currently keeps route paths stable while using virtual menu categories such as `/life` and `/inspect`.
* Existing social routes include salary, housing fund, pension, retirement date, time, and keycode tools.
* The working tree already contains WIP social tool pages and localization keys. Do not revert unrelated changes.

## Assumptions

* Preserve existing route paths for low-risk navigation changes.
* Prefer category label, icon, ordering, and sidebar affordance refinements over a route migration.
* Add small user-facing improvements rather than large policy/data integrations.

## Requirements

* Refine sidebar category naming so groups read consistently in Chinese and English.
* Keep mobile drawer, desktop expanded sidebar, collapsed sidebar, breadcrumbs, and active states consistent after the taxonomy change.
* Improve existing social/income tools with focused, useful feature or polish items.
* Improve performance without changing the Liquid Glass visual system, styling, or motion language.
* Keep all visible copy localized in `src/locales/cn.json` and `src/locales/en.json`.
* Avoid overwriting unrelated WIP changes.

## Acceptance Criteria

* [x] Sidebar categories use a more coherent taxonomy without changing route URLs.
* [x] The active category and active child are visually and semantically distinct.
* [x] The current route opens the correct category by default, and only one category expands at a time.
* [x] New or changed UI copy exists in both supported locales.
* [x] Performance optimizations avoid Liquid Glass visual changes.
* [x] `pnpm run lint`, `pnpm run typecheck`, and available tests pass.
* [x] Localhost verification covers the sidebar at the current route and the improved tool pages as far as available tools allow.

## Definition of Done

* Tests and static checks pass.
* Browser behavior is verified on the current local page.
* Any remaining risks or follow-up ideas are summarized for the user.

## Out of Scope

* Moving or renaming existing route paths.
* Adding external policy-data services.
* Completing a full site-wide information architecture redesign in this pass.
